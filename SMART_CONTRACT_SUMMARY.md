## Instructions

1. **initialize_config** - One-time protocol setup
2. **register_player** - Player registration
3. **deposit** - Deposit SOL into vault
4. **withdraw** - Withdraw SOL from vault
5. **set_exposure** - Set risk exposure (0-100%)
6. **resolve_cycle** - End cycle and calculate redistribution
7. **claim_redistribution** - Claim P2P redistribution

## Core Features

### Game Mechanics
- Cycle-based gameplay (100 slots = ~40 seconds)
- P2P redistribution (zero-sum game)
- Exposure system (0-100% risk)
- Real-time balance updates

### Security
- Anti-whale: 20% pool threshold, 50% penalty on excess
- Anti-sybil: 0.001 SOL minimum balance
- Cooldown: 10 slots between exposure changes
- Withdrawal protection: Can't withdraw with active exposure

### Economics
- 2% protocol fee (200 bps)
- Deposit limits: 0.001 - 1 SOL
- Fee collection to treasury vault
- Transparent redistribution math

## Math Logic

All calculations use fixed-point integer arithmetic:
- BPS_SCALE: 10,000 (basis points)
- EXPOSURE_SCALE: 100 (percentage)
- WEIGHT_SCALE: 1,000,000 (normalized weights)

Key functions:
- `calculate_fee()` - 2% protocol fee
- `calculate_exposed_value()` - balance × exposure%
- `normalized_weight()` - player share of pool
- `target_weight_with_anti_whale()` - whale penalty
- `calculate_redistribution_share()` - P2P redistribution
